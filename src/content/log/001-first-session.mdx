Use one entry per research or coding-agent session. A good entry should record the goal, the interaction pattern, surprising failures, useful prompts, and any artifacts worth preserving.<Sidenote>For example: did the agent misread a file, overfit to a prior assumption, discover a simpler design, or produce a useful test?</Sidenote>

## Session outline

1. Goal of the session.
2. Tools and agents used.
3. What happened.
4. Evidence, transcript excerpts, or code links.
5. Reflections for the next session.

<MarginNote>
Keep marginal notes short. If the comment becomes central to the argument, promote it to body text.
</MarginNote>
