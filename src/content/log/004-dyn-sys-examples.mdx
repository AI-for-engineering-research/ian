[Transcript](/sessions/004-dyn-sys-examples/)

- This session was about making some animations of well-known dynamical
  systems as examples to put on the project overview page. This turned out to
  be much easier than I expected! Once I got the agent on the right track for
  what I wanted, it mostly became another yes, yes, yes session.
- For the more complicated example (a reaction-diffusion PDE), I found an
  exact existing model to follow, and I could just [give that to the
  agent](/sessions/004-dyn-sys-examples/#entry-61).
- One thing you can [see in this
  session](/sessions/004-dyn-sys-examples/#entry-92) is the use of the
  [Backlog.md](https://github.com/MrLesk/Backlog.md) issue tracker that I
  like. This is really handy for more complex tasks: you get the agent to make
  tasks (they're like GitHub issues, but just live in files in your project)
  where it can store notes, a detailed description of what it needs to do, and
  so on. This means that even if you need to start over in a new session
  because of context issues, the agent knows what to do. You can even do what
  I did here and run the agent in [an AFK
  loop](/sessions/004-dyn-sys-examples/#entry-103), so that you get your
  dynamical systems animations built while you eat your dinner...
- I had [more permissions issues
  here](/sessions/004-dyn-sys-examples/#entry-97), because I was working in a
  Git worktree. That's always fun with sandboxes, but it was relatively easy
  to figure out this time round.
- A little iteration was needed for each of the three examples, and I did some
  manual experimentation with model parameters to get good visualizations, but
  it was overall a very quick and simple process.
